Data Science Analysis of The Joe Rogan Experience

Predicting the Likes/Dislikes ratio on Youtube videos using Youtube API data by Kaan Eyvaz.

SOURCE CODE

INTRODUCTION

This project aims to answer the question of whether you can predict the ratio of likes and dislikes of youtube videos. I have chosen to analyse Joe Rogan Experience videos to answer this question. The Joe Rogan Experience is an extremely popular podcast that hosts a variety of different guests. The type of guests and the topics covered on the podcast vary greatly and each guest individually garners a unique reaction from the audience.

THE DATA

The data was collected from the Joe Rogan Experience YouTube channel through the use of the YouTube data API. There are almost 1400 videos in the dataset and a total of 3.5million comments. I querired the API for video IDs of each episode, I then used these video IDs to query for the statistics, dates and comments which is in JSON format. The comments were obtained in a seperate file form the video statistics and dates. The comments were then processed through my own custom programmed NLP algorithm to apply a sentiment analysis for each individual comment. The sentiment analysis used a 'Bag of Words' model to evaluate the sentiment of each comment. Each comment was given a numbered label corresponding to which video the comment belonged.

THE NLP

Each comment is treated as a document and is then broken down into a bagh of words. Each word in the comment is then tokenised using the spaCy python library. Each word is lemmatised to get the root meaning of the word. A list of 144,000 words with sentiments scored attached to them were obtained free from SentiWords, a common method of compaing words to obtain the sentiment. After processing a comment, each word in the bag of words has been given a sentiment score, and the total sentiment score is average of these sentiments. The total magnitude is added up aswell from the sentiment of each word. After the comments were processed, the sentiment scores and magnitudes were grouped by the video they belonged to. I took the descriptive statistics of the grouped comments for each video and joined it to my dataFrame, along with the other columns of my YouTube video dataFrame.

Univariate Analysis

Right off the bat, we can see that the view count is quite evenly distributed with a fairly even amount of outliers, which is expected. It is interesting to see that category 4 has the highest number of likes, compared to the other categories which are all outliers. Conversely, category 0 has the highest number of dislikes, far more than any of the other categories. We can also see that low ratio videos have a higher magnitude.

Most of these histograms are visibly skewed to the right, except for the sentMean and magMean which don't have that much skew.

Bivariate Analysis

There is not much correlation of key variables except for the commentCount, likeCount and dislikeCount, which all seem to be correalated with the viewCount.

Basic Regression: Linear Regression

The best performance of the Linear Regression model is approximatley 0.24 R Squared. Not a strong correaltion at all.

Basic Classification: Logistic Regression

The Logistic Regression model also has a low Accuracy Score of approximatley 0.27

Improved Modelling

For the improved modelling, I used RandomForestRegressor for regression, and RandomForestClassifier and DecisionTree for classification. They were all run through GridSearchCV in order to find the best combination of parameters for the best training and testing result possible.

Improved Regression: Random Forrest Regessor

The Random Forest Regessor has a better R Squared value of approximatley 0.33 however this is still a low score

Improved Classifiication: DecisionTreeClassifier, RandomForestClassifier and GridSearchCV

The Random Forest Classifier and Decision Tree models both have an accuracy score of approximatley 0.29 which is higher than the Logisitic Regression model but still relativley low.

Future Improvemnt and Conclusion

In conclusion I have found that you cannot predict the likes/dislikes ratio with the current features. Moving forward I would be interested in making improvemnts to better my analysis. Among these improvemnts would be:

  • More data. With 1400 videos, i'd like to feed the algorithms more data to get a better picture of which features are important to predicting ratio.
  • More features. Labelling the data as female/male, perhaps taking into account time of day, day of the week, or the year.
  • Choosing a wider variety of videos. Because I focussed on a single channel, I feel there may be stronger results with a more diverse set of videos.
  • Choosing videos with a minimum of 50,000 comments. The best performing features were the sentiment scores. I feel that our models would perform better if they had more comment data to form the sentiment scores.
  • Predicting different features. I focussed on ratio for this study but had considered trying to predict the type of guest on the podcast or perhaps the gender of the guest, or political bias and such. There are many directions to could go in regards to the data set because I feel that The Joe Rogan experience is a diverse enough channel with lots of different things to analyse
